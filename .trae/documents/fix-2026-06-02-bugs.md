# 修复 Bug 计划（4 个问题）

## 0. 根因速览

| Bug | 根因 |
|---|---|
| 1. 第一次点刷题按钮无响应 | `PracticeView` 的 `modal` 是 `ref<ModalState \| null>(null)`，配合 `v-if="modal"` 渲染。子组件 `PracticeModal` 第一次挂载时 `props.show` 直接为 `true`（**不是** false→true 切换），`watch(() => props.show)` 不会触发；`onMounted` 又没兜底，所以 `doGenerate` 不被调用。第二次点时 modal 仍存在（`null → 新对象` 整体替换，但 inner `show: true → true` 实际是 false→true 由 v-if 触发）所以能跑通。 |
| 2. 主观题成功后 SSE 模态不自动隐藏 | `createSseContext.ask` 的成功路径 `return obj as T` **没有**重置 `isStreaming / showStream`，只有 `catch` 路径才重置。失败和取消时关闭，但成功时一直保留 streaming 状态。 |
| 3. 客观题 3 次重试都失败 | `OBJECTIVE_GENERATE_SCHEMA` 写 `options: 'string[]'`，但 AI 实际返回的 `options` 是 `[{label, text}, ...]` **对象数组**。`validateShape` 用 `typeof === 'string'` 逐元素检查 → 全部失败 → 抛错 → 触发 retry。3 次都失败。 |
| 4a. 两个 system message | `useAiCallRaw.streamChat` 已经 `prependSystem(messages)`，但 `createSseContext.ask` 收到 messages 后又 prepend 一次 system。**双重 prepend**。 |
| 4b. 整卷时附带的 .md 是某个具体 subject 的 | `usePracticeFlow.generate` 在 `scope === 'exam'` 时只把 `currentExam.subjects[0]` 当作 subject 传给 `buildPrompt`，`buildKnowledgeContext` 内部又 `findModule(subject.name)?.content` —— 结果把第一个 subject 的 .md 当成"整卷知识"塞给 AI。其它 8 个 subject 的 topics 完全没出现。 |
| 4c. 整卷出题失败 | 由 3 引起，options schema 不通过。 |

---

## 1. 修复设计

### 1.1 Bug 1：PracticeModal 首次挂载自动触发

**思路**：把"父→子"的"打开"事件从 `props.show` 单字段升级为"显式信号"——`props.show` 仍是显隐开关，但模态打开时由 `PracticeView` 通过 `props.openKey` 唯一 token 变化驱动 `doGenerate`。这样无论 modal 第一次创建还是后续开关，都能稳定触发。

**实现**：

- `PracticeView.vue`：
  - `modal` 改用常驻对象 `ref<ModalState>({ show: false, subject: 兜底, topic: 兜底, scope: 'topic' })`，**不再为 null**，这样 `v-if` 始终渲染（display:none 状态），子组件不会反复 mount/unmount。
  - 新增 `openKey = ref(0)`；`openTopic/openSubject/openExam` 自增 `openKey`。
  - 模板中给 `PracticeModal` 加 `:open-key="openKey"`，**不再**用 `v-model:show`（改为 `:show="modal.show" @update:show="modal.show = $event"`）—— 或者继续用 `v-model:show`，由 modal 自己控制显隐。
  - `openTopic` 内：先 `modal.value = { show: true, subject, topic, scope: 'topic' }`，**然后** `openKey.value++`。

- `PracticeModal.vue`：
  - 去掉 `watch(() => props.show)` 这一唯一触发源。
  - 改用 `watch(() => props.openKey, async (k) => { if (k > 0 && isNormalMode.value) { reset(); await doGenerate() } }) { immediate: true }`。
  - 这样父组件每次 `openKey++`，无论 `show` 是否变化，模态都会重新走一次 `reset + doGenerate`。

### 1.2 Bug 2：成功后关闭 SSE 模态

**思路**：成功路径里 `return` 前重置 `isStreaming / showStream`，并加 200ms 缓冲让用户看到最终输出（避免突兀消失）。给 `usePromptStore` 的 `'system'` prompt 留个 `'assistant'` 角色的"完成提示"位也可选；最简方案就是直接关闭。

**实现**：

- `createSseContext.ask`：
  ```ts
  for (let i = 0; i < retries; i++) {
    try {
      const full = await streamChat(...)
      const obj = extractJson(full)
      validateShape(obj, opts.schema)
      // ← 新增：成功后延迟关闭 SSE 模态
      setTimeout(() => {
        isStreaming.value = false
        showStream.value = false
      }, 200)
      return obj as T
    } catch (e) { ... }
  }
  ```

### 1.3 Bug 3：options schema 改成对象数组

**思路**：客观题 `options` 是 `[{label, text}, ...]`，schema 应该用 `JsonShape[]` 描述结构（label 必填 string、text 必填 string）。`json.ts` 的 `check` 函数已经支持 `Array.isArray(value)` 分支，所以无需修改校验器。

**实现**：

- `usePracticeFlow.ts`：
  ```ts
  const OBJECTIVE_GENERATE_SCHEMA: JsonShape = {
    topicId: 'string',
    topicName: 'string',
    single: {
      question: 'string',
      options: [{ label: 'string', text: 'string' }],   // ← 对象数组
      correctAnswer: 'string',
      explanation: 'string',
    },
    multiple: {
      question: 'string',
      options: [{ label: 'string', text: 'string' }],   // ← 对象数组
      correctAnswer: 'string[]',
      explanation: 'string',
    },
  }
  ```
- `llm/json.ts` 的 `check` 已经有 `if (Array.isArray(value))` 分支，会用 `value[0]` 作为 item shape 校验每个元素 → 通过。

### 1.4 Bug 4a：去重 system

**思路**：让 `useAiCallRaw` 不再 prepend system，纯做透传；由 `createSseContext.ask` 唯一负责 prepend。`useAiCallRaw.chat` 同步去掉 prepend（之前没人用，将来要用时再 prepend）。

**实现**：

- `useAiCallRaw.ts`：
  - 删除 `prependSystem` 函数及两处调用（`chat` 和 `streamChat` 中）
  - `buildBody` 直接接 `messages` 不变

### 1.5 Bug 4b+c：整卷出题改为附"整卷 subjects JSON"

**思路**：
- `usePracticeFlow.generate` 在 `scope === 'exam'` 时，**接收整个 exam 的所有 subjects**（而不是第一个）。
- `buildPrompt` 新增可选的 `extras.examSubjectsJson`，把 `exams.map(s => ({id, name, topics:[{id,name,keywords}]}))` 序列化后塞进 prompt。
- `buildKnowledgeContext` 在 `scope === 'exam'` 时不再返回某个 subject 的 .md，而是返回 `''` 或一句简短说明（让 prompt 占位符不尴尬），整卷知识上下文完全由 `extras.examSubjectsJson` 承担。
- 修默认 `objective-generate` prompt，把 `【出题范围】: 整卷综合（可涵盖该试卷任意科目的任意考点）` 后面追加 `【试卷所有可选科目与考点】:\n{examSubjectsJson}`（占位符替换）。

**实现**：

- `usePromptStore.ts`：
  - `buildKnowledgeContext` 当 `scope === 'exam'` 时返回 `''`（**不返回**第一个 subject 的 .md）
  - `buildPrompt(key, subject, topic, scope, extras)` 保持签名，新增 `extras.examSubjectsJson` 即可
- `usePromptStore.ts` 的默认 prompt `objective-generate` / `subjective-generate` 末尾追加：
  ```text
  【试卷所有可选科目与考点（仅在整卷演练时出现）】:
  {examSubjectsJson}
  ```
  并在 `buildPrompt` 的 replace 阶段处理。
- `usePracticeFlow.ts` 的 `generate`：
  ```ts
  async function generate(subject, topic, scope): Promise<...> {
    const extras: Record<string, string> = {}
    if (scope === 'exam') {
      const exam = examById(examId.value)
      if (exam) {
        const slim = exam.subjects.map(s => ({
          id: s.id, name: s.name,
          topics: s.topics.map(t => ({ id: t.id, name: t.name, keywords: t.keywords })),
        }))
        extras.examSubjectsJson = JSON.stringify(slim, null, 2)
      }
    }
    const prompt = buildPrompt(promptKey, subject, topic, scope, extras)
    ...
  }
  ```
- `buildPrompt` 的 placeholder replace：把 `\{examSubjectsJson\}` 替换为 extras 中的值（替换时如果整卷场景下占位符无值，避免输出 "undefined"；可以在 `replace` 后做一次空占位符过滤）。

### 1.6 顺带清理：失败信息更友好

- `validateShape` 抛出的 `single.options 应为 string[]` 来自根字段名 'single' 拼接出来的 path 'single.options'；bug 修复后这个错误消失，**不再**需要修错误信息。但为了让出错时更清楚，在 `createSseContext.ask` 的 `catch` 里加：
  ```ts
  lastErr = new Error(`第 ${i + 1} 次失败: ${e?.message}`)
  ```
  把每次重试的错误都吐给用户（最后一次的错误已经在 throw 里）。

## 4. 优雅性自检

- 触发器从"props 变化副作用"升级为"显式 openKey 信号"——`openKey` 每次 `++` 就是一个"再开一次"的语义命令，可观测、可测试、无 race condition。
- SSE 模态的成功关闭用 setTimeout 200ms 缓冲，给用户视觉余光，**不**突兀。
- schema 用 `[{label:'string',text:'string'}]` 描述对象数组，**比 `'string[]'` 强**（既校验数组性，又校验元素结构），复用现有 `validateShape` 不增加新分支。
- `useAiCallRaw` 退化为纯透传——谁 prepend system 由调用方决定，**职责清晰**。
- `buildKnowledgeContext` 在 exam scope 返回空，整卷上下文由 `extras.examSubjectsJson` 显式承担——**没有"特殊 scope 偷塞 .md"的隐式行为**。
- prompt 模板里 `{examSubjectsJson}` 是显式占位符，模板作者一眼能看出"整卷时会附加什么"。
